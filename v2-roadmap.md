# JSET v2 Roadmap (High-Level)

JSET v1 locks the honest baseline: the rigorous exceptional Jordan/E₆ representation scaffold (Standard Imported Theorems only), a typed mirror description, capacity as bookkeeping or Legendre (weights not derived unless data are independent), the family/observer count as the named conjecture **FAM**, corrected soldering requiring explicit complex selection, and all EFT layers as conditional on named assumptions. All claims are classified; the six open problems are stated with checkable success metrics.

**JSET v2** will be the first version in which one or more of the open problems (see `OPEN-PROBLEMS.md`) have been discharged (promoted to theorems, with the corresponding ledger assumptions removed or weakened) **or** in which it is explicitly shown that certain numbers must remain external inputs (with the framework's added value stated honestly).

## High-Level Criteria for v2

- At least one of the six open problems has a rigorous solution (or refutation) that is incorporated into the main ledger.
- The corresponding physical statements (e.g., "three families", "d=4 Lorentzian from the scaffold", specific capacity weights, particular EFT operators or RG coefficients) are reclassified from Conjecture/Assumption/CEC to Theorem or are explicitly labeled as remaining assumptions with justification.
- The parameter-to-observable economy and contact status are re-audited against the new ledger (retiring any claims that still rely on the old reverse-engineered elements).
- A short "what is new in v2 vs. ordinary E6 GUT model building" section is added (or the honest statement that no new quantitative relations yet appear).
- All public documents (README, any PRD companion, one-pager) are updated for the new classifications.
- Historical material remains isolated; the v1 ledger is the stable citation target for the scaffold itself.

## Suggested Attack Order (pragmatic)

1. **Open Problem 3 (Explicit family geometry / FAM)** — highest leverage for "three families" claims. A correct, finite or algebraic model with computable anti-invariant cohomology would immediately allow reclassification of the observer count.
2. **Open Problem 2 (Independent capacity data)** or **4 (Complex soldering selection)** — these control the weights and the spacetime signature, which propagate into almost every conditional layer.
3. **Open Problem 1 (Unified mirror category)** — restores some of the original generative narrative if successful; otherwise cements the typed description.
4. **Open Problem 5 (EFT action terms and RG coefficients)** — needed before any gravity or unification-scale claims can be called predictions rather than inputs.
5. **Open Problem 6 (Testable constraints)** — the ultimate test of whether the framework adds value beyond standard GUT model building with equivalent freedom.

Progress on any item can be incremental (e.g., a partial model for FAM that gives a definite integer even if not yet profinite).

## Relation to v1

v1 is deliberately conservative and self-contained. Nothing in the v1 deliverables claims to solve these problems. The `OPEN-PROBLEMS.md` and this roadmap exist so that future work has a clear, falsifiable target list rather than vague "more research is needed."

When v2 work begins, the first commit or note should reference this file and the corresponding sections of `sjet_true.tex`.

See also `CHANGELOG.md` (v1.0 entry) and the global summary theorems in `sjet_true.tex` ("What SJET_true is" and "What SJET_true is not").
